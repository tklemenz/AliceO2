96 //
97 // DataCompression tools
98 //
99 // The Alphabet: for every parameter to be stored in a compressed format, there
100 // is a range of valid symbols referred to be an alphabet. The DataCompression
101 // framework requires an implementation of an alphabet which is then used with
102 // the probability model and codecs. The alphabet class needs to implement a
103 // check function (isValid), and a forward iterator together with begin and end
104 // function to hook it up to the framework.
105 //
106 // Default Alphabet implementations: most of the alphabets will describe a range
107 // of contiguous values, which is implemented by the class ContiguousAlphabet.
108 // Two specializations describe symbols in a range from 0 to a max value and
109 // within a certain bit range, respectively
110 // ZeroBoundContiguousAlphabet and BitRangeContiguousAlphabet
111 
112 /******************************************************************************
113  * @class ExampleAlphabet
114  * @brief An example for an Alphabet definition.
115  *
116  * A class definition for an alphabet. Note that the functions are only defined,
117  * not implemented. The class is required to support the following functions to
118  * hook it up onto the DataCompression framework:
119  * - isValid validity of a value
120  * - getIndex get index from value
121  * - getSymbol get symbol from index
122  * - forward iterator class to walk through elements of alphabet
123  * - begin start iteration over elements
124  * - end end marker for iteration
125  *
126  * The alphabet has to provide an index for the symbol range such that the

129  */

186 /******************************************************************************
187  * Definition of a contiguous alphabet
188  *
189  * The contiguous alphabet defines integer number elements of the specified
190  * type in a given range.
191  *
192  * TODO: the mpl string length is limited and can be configured with the
193  * define BOOST_MPL_LIMIT_STRING_SIZE (somehow the number of allowed
194  * template arguments is given by this number divided by four)
195  */

197  typename T,
198  T _min,
199  T _max,
200  typename NameT = boost::mpl::string<'U','n','n','a','m','e','d'>::type
201  >
202 class ContiguousAlphabet {
203 public:
204  ContiguousAlphabet() {}
205  ~ContiguousAlphabet() {}

213  static bool isValid(value_type v) {
214  return v >= _min && v <= _max;

222  static unsigned getIndex(value_type symbol) {
