1 QC
2 =======
3 
4 __This is not the Quality Control.__ The Quality Control code is in
5 [this repository](https://github.com/AliceO2Group/QualityControl).
6 
7 This is a merging prototype for AliceO2 project.
8 It is under the QC directory for historical reasons but
9 will be renamed to _DataMerger_ when the code is clean and general.
10 
11 # Architecture
12 This project consists of four modules that described below :
13 - Producer
14 - Merger
15 - Viewer
16 - MetricsExtractor
17 
18 ### Producer - produces Quality Control objects
19 Required arguments:
20 
21  - TH1F: DDS topology property id, device id, TH1F option, object name, object title, buffer capacity, number of bins
22 
23  - TH2F: DDS topology property id, device id, TH2F option, object name, object title, buffer capacity, number of bins
24 
25  - TH3F: DDS topology property id, device id, TH3F option, object name, object title, buffer capacity, number of bins
26 
27  - THnF: DDS topology property id, device id, THnF option, object name, object title, buffer capacity, number of bins
28 
29  - TTree: DDS topology property id, device id, TTree option, object name, object title, buffer capacity, number of bins, number of branches, number of entries in each branch
30 
31 where:
32 
33  - DDS topology property id: id of the topology property holding merger address (e.g. mergerAddr)
34  - device id: id of the device (e.g. mergerAddr)
35  - option: one of the option of object type to produce (TH1F, TH2F, TH3F, THnF or TTree)
36  - object name: name of the produced objects (e.g. histogramName)
37  - object title: title of the produced objects (e.g. histogramTitle)
38  - buffer capacity: capacity of the outpu buffer (e.g. 100)
39  - number of bins: number of bins in produced QC data object (e.g. 1000)
40  - number of branches: number of branches in TTree QC object (e.g. 4)
41  - number of entries in each branch: number of entries in each branch in TTree QC object (e.g. 1000)
42 
43 Run example for histogram:
44 
45 ~~~~~~~{.sh}
46 
47 runQCProducerDevice mergerAddr deviceID TH1F histogramName histogramTitle 100 1000
48 ~~~~~~~
49 
50 ### Merger - merges received objects.
51 Required arguments:
52 
53  - DDS topology property id: id of the topology property holding merger address (e.g. mergerAddr)
54  - device id: id of the device (e.g. deviceID)
55  - required number of objects with the same name to merge (e.g. 100)
56  - merger input TCP port (e.g. 5016)
57  - input buffer capacity (e.g. 500000)
58  - output address with TCP port number (e.g. tcp://login01.pro.cyfronet.pl:5004)
59 
60 Run example:
61 
62 ~~~~~~~{.sh}
63 
64 runQCMergerDevice mergerAddr deviceID 100 5016 500000 tcp://login01.pro.cyfronet.pl:5004
65 ~~~~~~~
66 ### Viewer - provides visualization of merged objects.
67 Optional arguments:
68 
69  - drawing option: drawing option passed to Draw function of a QC object (e.g. branchtoDrawName)
70 
71 Run example:
72 
73 ~~~~~~~{.sh}
74 
75 runQCViewerDevice branchToDrawName
76 ~~~~~~~
77 ### MetricsExtractor - used for metrics extraction from nodes.
78 Sends DDS custom commands to all of the nodes in a topology. It accepts responses as a json structures with valid custom command name.
79 
80 Required arguments:
81 
82  - output file suffix name: suffic to be added to out file name of nodes metrics (e.g. metricSuffix)
83 
84 Run example:
85 
86 ~~~~~~~{.sh}
87 
88 runQCMetricsExtractor metricSuffix
89 ~~~~~~~

91 # Build
92 
93 ### Prerequisites
94 0. Install AliceO2 and DDS software.
95 1. Set the environment variable SIMPATH to your FairSoft installation directory.
96 2. Set the environment variable FAIRROOTPATH to your FairRoot installation directory.
97 
98 It is a good practice to run config.sh script from AliceO2 build directory to
99 set all others variables such as PATH etc.
100 
101 ### Compilation
102 Go to build folder of AliceO2 software
103 ~~~~~~~
104 cmake ../
105 cd Utilities/QA
106 make all
107 ~~~~~~~
108 
109 # Test
110 All modules are provided with unit tests written in BOOST test framework. Each module has tests in "Tests" subdirectory.
111 To run all unit tests type `ctest`
112 
113 # Run
114 See this page: http://dds.gsi.de/doc/nightly/RMS-plugins.html#slurm-plugin to execute system with DDS SLURM plug-in.
115 
116 Mergers and Producers have to be run with DDS topology. MetricsExtractor and Viewer should be run with bash shell.
